# Внешняя компонента

Высокопроизводительное асинхронное решение для параллельной работы с сетью (HTTP/HTTPS) в **1С:Предприятии 8.3**. Проект состоит из внешней компоненты, написанной на **Rust** (Native API), и готового расширения конфигурации 1С, построенного по модульному принципу SOLID.

## 🚀 Ключевые возможности

- **Асинхронный движок на Rust**: Архитектура на базе `tokio` и `reqwest`.
- **Гарантия доставки (RabbitMQ-style ACK)**: Двухфазное чтение кэша (`At-Least-Once`), очистка памяти только после `ПодтвердитьПолучение` в 1С.
- **Защита от OOM (Gzip на диск)**: Перехват «тяжелых» ответов (>20 МБ), сжатие и сохранение на диск.(В процессе)
- **Умные ретраи**: Повторы запросов с `tokio-retry2`.
- **Strict Mode**: Проверка и валидация данных перед отправкой в 1С.
- **Сетевая безопасность**: Поддержка CA, игнорирование SSL, mTLS (PEM).(Не тестировалось)

## 📂 Структура репозитория

- `/rust` — Исходный код внешней компоненты на Rust.
- `/1c` — Расширение конфигурации для 1С.

## 💻 Спецификация API и Структуры Данных

Обмен между 1С и внешней компонентой происходит строго через сериализованные JSON-пакеты. Благодаря атрибутам `#[serde(default)]` на стороне Rust, компонента прощает 1С-разработчикам отсутствие необязательных полей и автоматически подставляет безопасные дефолты.

### 1. JSON-манифест настроек движка (Метод `Настроить`)
Передается один раз при старте сессии (или динамически при изменении параметров в 1С):
```json
{
  "max_rows": 1000,                  // Макс. размер очереди задач в ОЗУ
  "secs_timeouts": 30,               // Таймаут HTTP-запросов (в секундах)
  "danger_accept_invalid_certs": true // Игнорировать ошибки SSL (для тест-контуров)
}
```

### 2. JSON-манифест пакета задач (Метод `ДобавитьЗадачу`)
1С может выплюнуть массив из тысяч таких объектов за один вызов:
```json
{
  "tasks": [
    {
      "task_id": "req_invoice_99812", // Уникальный ID задачи в 1С (Обязательно)
      "service_name": "Касса",         // Имя сервиса для фильтрации результатов
      "url": "https://site.ru", // Целевой URL (Обязательно)
      "method": "POST",                // HTTP-метод (GET, POST, PUT и др.)
      "headers": {                     // Произвольные HTTP-заголовки
        "Authorization": "Bearer token_123",
        "Content-Type": "application/json"
      },
      "payload": "{\"amount\": 1500}"  // Тело запроса (строка)
    }
  ]
}
```

### 3. JSON-структура ответа ВК (Метод `ЗабратьРезультаты`)
Возвращает массив выполненных задач. Обратите внимание на архитектуру защиты от переполнения памяти (OOM): если ответ от сервера весит более 20 МБ, Rust-ядро сжимает его в Gzip, сбрасывает на диск кластера и возвращает метаданные с путем к файлу:

```json
[
  {
    "task_id": "req_invoice_99812",
    "service_name": "Касса",
    "status_code": 200,               // HTTP статус-код ответа
    "response_body": "{\"status\":\"ok\"}", // Тело ответа (если размер < 20 МБ)
    "error": null,                    // Текст сетевой ошибки (если упал DNS/TCP)
    "unix_timestamp": 1719876543,
    // Метаданные защиты от OOM (если размер > 20 МБ):
    "is_saved_to_disk": false,        
    "absolute_path": null             // Будет содержать путь к .json.gz при тяжелом ответе
  }
]
```

## 📋 Пример интеграции в 1С (BSL)

Вся сложная асинхронная магия Rust для разработчика 1С инкапсулирована в простые линейные вызовы. Ниже приведен пример **высоконадежного транзакционного обмена** (паттерн At-Least-Once / Гарантия доставки), который исключает потерю данных при сбоях 1С.

### 1. Быстрый заброс задач в конвейер (без зависания интерфейса)
```bsl
МассивЗадач = Новый Массив;

Для Каждого СтрокаТовара Из ТаблицаВыгрузки Цикл
    Задача = Новый Структура;
    Задача.Вставить("task_id",      Строка(СтрокаТовара.Номенклатура.УникальныйИдентификатор()));
    Задача.Вставить("service_name", "ВыгрузкаСайт");
    Задача.Вставить("url",          "https://mysite.com");
    Задача.Вставить("method",       "POST");
    Задача.Вставить("payload",      НВК_Сериализация.ДанныеВСтрокуJSON(СтрокаТовара));
    
    МассивЗадач.Добавить(Задача);
КонецЦикла;

// Включаем Strict Mode (Истина) — 1С сама проверит пакет, исправит некритичные пропуски
// ВК мгновенно примет массив в ОЗУ на "зеленые потоки" Tokio и сразу вернет управление!
УспехОтправки = НВК_СетевойДвижок.ДобавитьПакетЗадачВКонвейер(МассивЗадач, Истина);
```

### 2. Регламентный забор ответов и транзакционное ACK-подтверждение
```bsl
// Вызывается фоновым регламентным заданием по таймеру
Процедура ОбработатьОчередьОтветовОтВК() Экспорт
    
    // Забираем из ВК готовые ответы по нашему сервису (не более 500 штук за раз)
    // В этот момент данные НЕ удаляются из памяти Rust движка!
    ОтветыВК = НВК_СетевойДвижок.ЗабратьРезультатыЗапросов("ВыгрузкаСайт", 500);
    
    Если ОтветыВК.Количество() = 0 Тогда
		Возврат; // Ждем следующего тика регламента
	КонецЕсли;
    
    МассивIDДляОчистки = Новый Массив;
    
    // Открываем транзакцию 1С для гарантированной записи данных в базу
    НачатьТранзакцию();
    
    Попытка
        Для Каждого Ответ Из ОтветыВК Цикл
            
            ИдЗадачи   = Ответ.task_id;
            КодСтатуса = Ответ.status_code;
            
            Если ЗначениеЗаполнено(Ответ.error) Тогда
                // Логируем сетевую ошибку (упал интернет, домен не найден)
                ЗаписьЖурналаРегистрации("НВК_Обмен", УровеньЖурналаРегистрации.Ошибка,,, 
                    "Задача " + ИдЗадачи + " упала: " + Ответ.error);
                МассивIDДляОчистки.Добавить(ИдЗадачи);
                Продолжить;
            КонецЕсли;
            
            // ПРЕДОХРАНИТЕЛЬ OOM: Проверяем, не сброшен ли тяжелый файл на диск
            Если Ответ.Property("is_saved_to_disk") И Ответ.is_saved_to_disk = Истина Тогда
                // [В ПРОЦЕССЕ РЕАЛИЗАЦИИ]: 
                // Сюда пойдет логика: ЧтениеZipФайла -> Поточное чтение JSON с диска,
                // чтобы не грузить гигабайты текста в одну переменную 1С.
                ПутьКАрхиву = Ответ.absolute_path; 
                МассивIDДляОчистки.Добавить(ИдЗадачи);
                Продолжить;
            КонецЕсли;
            
            // Обычная обработка легкого ответа из ОЗУ
            Если КодСтатуса = 200 Тогда
                // Парсим response_body и записываем данные в регистры/документы 1С
                ЗаписатьРезультатВДокумент(ИдЗадачи, Ответ.response_body);
            КонецЕсли;
            
            // Накапливаем ID успешно обработанных задач
            МассивIDДляОчистки.Добавить(ИдЗадачи);
            
        КонецЦикла;
        
        // Фиксируем транзакцию в базе данных 1С!
        ЗафиксироватьТранзакцию();
        
    Исключение
        // Если база 1С заблокировалась или документ не провелся — откатываем транзакцию.
        // ACK-сигнал НЕ будет отправлен в Rust, и при следующем тике регламента 
        // мы снова прочитаем эти же задачи из ВК без потери данных!
        ОтменитьТранзакцию();
        ЗаписьЖурналаРегистрации("НВК_Обмен", УровеньЖурналаРегистрации.КритическаяОшибка,,, 
            "Сбой транзакции 1С: " + ОписаниеОшибки());
        Возврат;
    КонецПопытки;
    
    // RabbitMQ-style ACK-рукопожатие: 
    // Только ПОСЛЕ успешной фиксации транзакции в 1С командуем Rust окончательно вычистить ОЗУ
    НВК_СетевойДвижок.ПодтвердитьПолучениеЗадач(МассивIDДляОчистки);
    
КонецПроцедуры
```
